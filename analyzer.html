Analizador de ensamblador x86

Pega instrucciones (sintaxis Intel, una por linea). Para cada una se busca su forma en la base de datos y se reporta lo que hace y su coste en la microarquitectura elegida.
Como funciona el analisis

Para cada instruccion (sintaxis Intel, una por linea; se ignoran etiquetas nombre: y comentarios ; # //) se identifica el mnemonico, se clasifica cada operando (registro y su ancho, memoria — el ancho se toma del prefijo qword ptr [rax] o del otro operando —, inmediato) y se localiza su forma en la base de datos.

Del bloque completo se calculan las dos cotas que gobiernan el tiempo de ejecucion en la microarquitectura elegida:

throughput
Σ del throughput reciproco de cada instruccion: los ciclos si el bloque estuviera limitado por los puertos de ejecucion (instrucciones independientes, encauzadas).
latencia (camino critico)
se construye el grafo de dependencias reales entre instrucciones (registros — con sus alias de ancho —, flags y memoria) y se recorre el camino mas largo: los ciclos si el bloque estuviera limitado por las cadenas de dependencia.

La estimacion del bloque es el mayor de los dos, y se indica cual es el cuello de botella. La eliminacion de movimientos (mov registro-registro a 0 ciclos) y las latencias por camino operando→operando se tienen en cuenta.
